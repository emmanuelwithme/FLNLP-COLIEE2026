# Legal Case Retrieval

[中文](README.md) | [Back to Root](../README.md)

This directory contains the full COLIEE Task 1 workflow and is the primary working area of this repository: preprocessing, BM25, dense retrieval, LightGBM reranking, and submission export.

## Directory Layout

- `pre-process/`: raw-file processing, label split, and scope JSON generation.
- `lexical models/`: BM25 / QLD indexing and search.
- `modernBert-fp/`: dense encoder training, embeddings, and ranking.
- `lightgbm/`: feature generation, `LGBMRanker` training, and cutoff post-processing.
- `lcr/`: shared Task 1 modules.
- `modernBert-fp-chunkAgg/`: chunk-aggregation experiment path.
- `modernBert/fine_tune/`: BM25 hard-negative generation and additional training utilities.

## Dataset Layout

The default structure is:

```text
coliee_dataset/task1/<YEAR>/
  task1_train_files_<YEAR>/
  task1_test_files_<YEAR>/
  task1_train_labels_<YEAR>.json
  task1_test_no_labels_<YEAR>.json
```

Common root variables:

- `COLIEE_TASK1_YEAR`
- `COLIEE_TASK1_ROOT`
- `COLIEE_TASK1_DIR`

## Workflow

### 1. Preprocessing

```bash
bash run_pre_finetune.sh
```

This step creates:

- `summary/`
- `processed/`
- `task1_train_labels_<YEAR>_train.json`
- `task1_train_labels_<YEAR>_valid.json`
- `train_qid.tsv`
- `valid_qid.tsv`
- `lht_process/BM25/`
- `lht_process/modernBert/finetune_data/`
- `lht_process/modernBert/query_candidate_scope.json`

### 2. Dense encoder training

```bash
python "Legal Case Retrieval/modernBert-fp/fine_tune/fine_tune.py"
```

If you already have a checkpoint, set these in `.env` or the shell:

- `TASK1_MODEL_ROOT_DIR`
- `TASK1_BASE_ENCODER_DIR`
- `TASK1_RETRIEVAL_MODEL_NAME`

The downstream steps will reuse the same settings.

### 3. Train / valid retrieval check

```bash
bash run_train_valid_inference_eval.sh
```

This step writes:

- train / valid BM25 rankings
- dense embeddings
- dense dot / cosine rankings
- train / valid metric summaries

### 4. Test retrieval and submission

```bash
bash run_test_retrieval.sh
```

This step writes:

- `processed_test/`
- `test_qid.tsv`
- test BM25 rankings
- test dense embeddings
- BM25 submission
- dense submission

### 5. LTR feature generation, training, and fixed top-k export

```bash
bash run_ltr_feature_train_valid_test.sh
```

This step writes:

- `train_features.csv`
- `valid_features.csv`
- `test_features.csv`
- `valid_predictions_raw.csv`
- `test_predictions_raw.csv`
- fixed top-k submission

### 6. Cutoff search and final submission

```bash
bash run_ltr_cutoff_postprocess.sh
```

This step reads the prediction CSVs generated by `run_ltr_feature_train_valid_test.sh` and writes:

- `cutoff_search/cutoff_summary.json`
- `cutoff_search/best_overall/test_submission_best_mode.txt`
- `COLIEE_LTR_FINAL_SUBMISSION_PATH`

## Configuration

The repo-root wrappers read `.env`, and shell variables override `.env` for a single run.

Most frequently adjusted variables:

- data: `COLIEE_TASK1_YEAR`, `COLIEE_TASK1_DIR`, `TASK1_TRAIN_RAW_DIR`, `TASK1_TEST_RAW_DIR`
- dense: `TASK1_MODEL_ROOT_DIR`, `TASK1_BASE_ENCODER_DIR`, `TASK1_RETRIEVAL_MODEL_NAME`
- test export: `TASK1_SUBMISSION_TOPK`, `TASK1_BM25_RUN_TAG`, `TASK1_EMBED_RUN_TAG`
- LTR: `COLIEE_LTR_OUTPUT_DIR`, `COLIEE_LTR_VALID_SCOPE_PATH`, `COLIEE_LTR_TEST_SCOPE_PATH`, `COLIEE_LTR_CUTOFF_CONFIG_JSON`
